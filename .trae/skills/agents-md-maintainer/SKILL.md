# AGENTS.md 维护（非业务错误）

作用与触发
- 目的：把可复现的语法错误、部署错误、系统性错误的“原因→修正→验证”沉淀到 AGENTS.md，降低重复犯错。
- 触发：AI 出错后、或用户明确要求记录；遇到业务逻辑实现问题不记录。

收录范围
- 语法错误：语法、类型、lint、API 使用方式错误。
- 部署错误：构建、打包、发布、环境、路径、路由、资源 404 等。
- 系统性错误：跨模块/工具链配置不一致、约定不遵循、通用流程缺陷。
- 排除：功能/需求理解偏差、业务逻辑实现细节。

记录模板
- 标题：类别 | 简述
- 日期：
- 场景与症状：
- 根因：
- 修正：
- 验证：
- 相关约束/约定：

操作步骤
- 打开仓库根目录的 AGENTS.md。
- 定位到“错误记录（Syntax/Deploy/System）”章节；若不存在则创建该章节。
- 采用“记录模板”追加条目，保持简洁、可执行、可复现。
- 如已有相同问题，合并为单条并更新更优修正与验证。
- 使用项目约定术语与路径；避免贴出敏感信息。

合并与去重规则
- 同一错误来源与修正相同则合并。
- 优先保留更清晰、更短、更通用的描述。
- 变更路由/构建基线等关键约定时，同步到对应“关键配置/约定”章节。

示例
- 标题：部署 | GitHub Pages 路由 404
- 日期：2026-03-27
- 场景与症状：项目页刷新 404，静态资源路径错误。
- 根因：vite.config.js base 未设置为 /prompt-assembly/；路由使用 BrowserRouter。
- 修正：设置 base=/prompt-assembly/；改用 HashRouter；通过 Actions 发布 dist。
- 验证：本地 npm run build 检查 dist；线上访问 GitHub Pages 资源加载自 assets/。
- 相关约束/约定：遵循部署文档；切换发布方式后确认 Pages Source。
